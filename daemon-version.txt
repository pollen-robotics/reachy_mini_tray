# Pinned reachy-mini (daemon) version baked into the uv-trampoline sidecar at
# build time. This is the single source of truth for which daemon version a
# tray release ships: each release pins a known-good version here, and the
# trampoline detects the spec change on the next launch and upgrades the venv
# in place (see `needs_upgrade` / `upgrade_venvs` in uv-wrapper).
#
# Bump this in a dedicated commit to move the whole app to a newer daemon.
#
# Format: a single bare PyPI version (e.g. 1.8.4). Blank lines and lines
# starting with '#' are ignored. Leave empty to fall back to "latest from
# PyPI" (non-deterministic - not recommended for releases).
#
# Overrides (do NOT edit this file for them): the REACHY_MINI_VERSION /
# REACHY_MINI_SOURCE env vars and a positional branch arg to build-sidecar.sh
# still take precedence, for local dev and branch builds.
1.8.4
